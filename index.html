My Core Beliefs and Coding Practices

Keep it Simple

Subtle animations, clean UI/UX, necessary content, simple code structure.I modularise code into smaller chunks/methods and ensure all the processes are non-blocking. This result in smooth and snappy UI.

Follow Standards

I try to follow all the agile standards and use the best availble frameworks and libraries. Refactoring and optimising the code to ensure scalability and efficiency is one of my top priority.

Relax

Frameworks / Libraries

Frameworks and libraries form the basis of a project. Its very crucial to choose the right framework and work with trusted feature rich libraries. A good framework promotes industry leading coding standards and provides us with enough flexibility to keep up with new feature requirements.

Javascript is my forte.

NodeJS

Node.js is a JavaScript runtime built on Chrome's V8 JavaScript engine. I prefer to use it for creating desktop app / as backend server / Console app.

ElectronJS

Electron allows for the development of desktop GUI applications using web technologies. Its simple, fast and affordable. It gets the work done really well. Nothing beats the native apps though.

ReactJS

React is a JavaScript library for building user interfaces. Its my go to framework for developing fast snappy Webapp, PWA and UI which requires complex user interaction.

Laravel

Laravel is a web application framework with expressive, elegant syntax. Laravel is probably the best framework to develop a robust backend for high performant scalable sites and APIs.

Flutter

Flutter is Google’s UI toolkit for building beautiful, natively compiled applications for mobile, web, and desktop from a single codebase. Its my go to framework for ios and android apps.

GraphQL

GraphQL is a query language for APIs and provides a complete and understandable description of the data in your API, gives clients the power to ask for exactly what they need and nothing more.
